# Frequently Asked Questions

## Getting Started

### What is NikCLI?

NikCLI is an AI-powered CLI tool for software development. It features:

- **Multi-provider AI support**: Claude, GPT, Gemini, Ollama
- **26+ built-in agents** for specialized tasks
- **150+ slash commands** for all development workflows
- **VM containers** for isolated development
- **Web3 integration** for blockchain development

### System Requirements

**Minimum:**
- Node.js 18+
- 4GB RAM
- Internet connection

**Recommended:**
- Node.js 20+
- 8GB RAM
- Fast internet

### Supported Package Managers

- npm
- yarn
- pnpm
- bun

## Agents

### Available Agents

| Agent | Purpose |
|-------|---------|
| `universal-agent` | All-purpose (70+ capabilities) |
| `vm-agent` | Secure virtualized execution |
| `react-expert` | React & Next.js |
| `backend-expert` | API & databases |
| `devops-expert` | CI/CD & infrastructure |
| `code-review` | Code quality |
| `autonomous-coder` | Fully autonomous |
| `frontend-expert` | UI/UX development |
| `vue-expert` | Vue.js |
| `optimization-expert` | Performance |

### How to Use Agents

```bash
# List agents
/agents

# Use an agent
/agent react-expert "Create a login form"

# Autonomous mode
/auto "Build a REST API"

# Parallel execution
/parallel "react,backend" "Create full-stack app"
```

## Operating Modes

### Available Modes

| Mode | Description | Command |
|------|-------------|---------|
| **Default** | Conversational with auto-todos | `/default` |
| **Plan** | Structured planning | `/plan [task]` |
| **VM** | Container development | `/vm` |

### Switching Modes

```bash
/default    # Default mode
/plan       # Planning mode
/vm         # VM mode
```

## Configuration

### Configuration Storage

Configuration is stored in `~/.nikcli/config.json`

### Key Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `temperature` | 1.0 | Model creativity |
| `maxTokens` | 6000 | Max output tokens |
| `securityMode` | safe | Security level |
| `approvalPolicy` | moderate | Approval strictness |

### Environment Variables

```bash
# AI Providers
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."

# Features
export NIKCLI_QUIET_STARTUP=true
export NIKCLI_SEMANTIC_CACHE=true
export NIKCLI_AGENT_MEMORY=true

# Caching
export REDIS_URL="redis://..."
```

## Security

### Security Modes

| Mode | Description |
|------|-------------|
| `safe` | Maximum security, all approvals required |
| `default` | Balanced, risky operations need approval |
| `developer` | Reduced restrictions |

### API Key Storage

API keys are stored securely with AES-256-GCM encryption in `~/.nikcli/config.json`.

### Approval System

The approval system requires confirmation for:
- File write operations
- Command execution
- Network access
- Configuration changes

## Troubleshooting

### NikCLI Not Responding

```bash
# Check status
/status

# View debug info
/debug

# Clear and restart
/clear
/exit
nikcli
```

### API Key Issues

```bash
# Check configuration
/config

# Re-set API key
/set-key anthropic sk-ant-...

# Test connection
/debug
```

### Memory Issues

```bash
# Check cache
/cache stats

# Clear cache
/cache clear

# Compact memory
/compact
```

### VM Issues

```bash
# List VMs
/vm-list

# Check health
/vm-health

# View logs
/vm-logs <id>

# Restart VM
/vm-stop <id>
/vm-create
```

### Reducing Costs

```bash
# Use efficient models
/model claude-3-haiku
/model gpt-4o-mini

# Enable caching
/redis-enable
/cache stats

# Use local models
/set-key ollama http://localhost:11434

```
