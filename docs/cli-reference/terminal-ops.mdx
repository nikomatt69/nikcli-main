## Overview

NikCLI provides powerful terminal operation capabilities that allow you to execute system commands, manage processes, and interact with your development environment directly from the chat interface. All terminal operations include built-in security policies and approval workflows.

## Terminal Commands

### `/run` - Execute Commands

Execute terminal commands with built-in security and approval workflows.

<Tabs>
  <Tab title="Basic Usage">
    ```bash # Basic command execution /run ls -la /run pwd /run whoami #
    Development commands /run npm install /run npm test /run npm run build # Git
    operations /run git status /run git add . /run git commit -m "Update
    documentation" ```
  </Tab>

{" "}
<Tab title="Advanced Options">
  ```bash # Execute with specific working directory /run --cwd=/path/to/project
  npm test # Execute with environment variables /run --env NODE_ENV=production
  npm run build # Execute with timeout (in seconds) /run --timeout=60 npm run
  build # Execute in background /run --background npm run dev # Execute with

    container-name ```
  </Tab>
</Tabs>

### `/sh` - Shell Alias

Convenient alias for `/run` command with identical functionality.

```bash
# These commands are equivalent
/sh npm test
/run npm test

# Shell scripting
/sh "cd src && find . -name '*.ts' | wc -l"
/sh "grep -r 'TODO' src/ | head -10"
```

### `/ps` - Process List

Display running processes with filtering and sorting options.

```bash
# List all processes
/ps

# List processes by user
/ps --user $(whoami)

# List processes by name pattern
/ps --name node
/ps --pattern "npm.*"

# List with detailed information
/ps --detailed
/ps --tree
/ps --sort cpu
/ps --sort memory
```

## Security Policies

### Command Approval

NikCLI implements multi-level command approval based on risk assessment:

<Tabs>
  <Tab title="Safe Commands">
    **Automatically Approved:** ```bash ls, pwd, whoami, cat, grep, find npm
    test, npm run build, npm install git status, git log, git diff node
    --version, npm --version ```
  </Tab>

{" "}
<Tab title="Moderate Risk">
  **Requires Confirmation:** ```bash rm, mv, cp (with multiple files) npm
  publish, npm run deploy git push, git merge, git rebase docker build, docker
  run chmod, chown (limited scope) ```
</Tab>

{" "}
<Tab title="High Risk">
  **Requires Explicit Approval:** ```bash rm -rf, sudo commands system
  configuration changes network operations (curl, wget with write) database
  operations production deployments ```
</Tab>

  <Tab title="Blocked Commands">
    **Never Allowed:** ```bash format, fdisk, dd system shutdown/reboot kernel
    modifications security bypass attempts malicious patterns ```
  </Tab>
</Tabs>

### Security Modes

Configure security levels for different development scenarios:

<AccordionGroup>
  <Accordion title="Safe Mode (Default)">
    ```bash /safe-mode ``` **Characteristics:** - Maximum security restrictions
    - Approval required for all file modifications - No system-level commands -
    Sandboxed execution environment - Comprehensive audit logging
  </Accordion>

{" "}
<Accordion title="Developer Mode">
  ```bash /dev-mode enable 60 # Enable for 60 minutes ``` **Characteristics:** -
  Reduced security restrictions - Streamlined approval process - Access to
  system commands - Enhanced development tools - Time-limited activation
</Accordion>

  <Accordion title="Custom Security Policies">
    ```bash /security set approval-mode selective /security set
    command-execution restricted /security set file-operations safe ```
    **Options:** - `approval-mode`: none, selective, strict -
    `command-execution`: blocked, restricted, allowed - `file-operations`:
    readonly, safe, unrestricted
  </Accordion>
</AccordionGroup>

## Command Output Management

### Streaming Output

Real-time command output with progress tracking:

```bash
# Stream output in real-time
/run --stream npm run build
/run --stream docker build .
/run --stream npm test -- --watch

# Control output buffering
/run --buffer-lines 100 npm run build
/run --no-buffer tail -f log.txt
```

### Output Formatting

Control how command output is displayed:

<Tabs>
  <Tab title="Display Options">
    ```bash # Syntax highlighting for common outputs /run --highlight cat
    package.json /run --highlight git log --oneline # Compact output format /run
    --compact npm list /run --compact docker ps # Verbose output with metadata
    /run --verbose npm run build /run --debug docker-compose up ```
  </Tab>

  <Tab title="Output Processing">
    ```bash # Filter output /run --grep "error" npm run build /run --filter
    "warning" npm test # Limit output lines /run --head 20 npm run build /run
    --tail 50 docker logs container # Save output to file /run --save build.log
    npm run build /run --append test.log npm test ```
  </Tab>
</Tabs>

### Output History

Access and manage command output history:

```bash
# Show recent command outputs
/output list
/output history

# View specific command output
/output show 123
/output replay "npm run build"

# Search output history
/output search "error"
/output grep "warning"

# Export output
/output export build-logs.txt
/output save 123 command-output.txt
```

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Command Not Found">
    **Problem:** Command execution fails with "command not found" **Solutions:**
    ```bash # Check command availability /which command-name # Check PATH
    environment /env PATH # Use full path /run /usr/local/bin/node --version #
    Install missing commands /run which brew && brew install missing-package ```
  </Accordion>

{" "}
<Accordion title="Permission Denied">
  **Problem:** Command fails due to insufficient permissions **Solutions:**
  ```bash # Check file permissions /run ls -la file-or-directory # Use
  appropriate permissions /run chmod +x script.sh # Run with proper user context
  /run --user $(whoami) command # Use sudo when necessary (with approval) /run
  sudo command # Requires approval ```
</Accordion>

{" "}
<Accordion title="Process Hanging">
  **Problem:** Commands hang or become unresponsive **Solutions:** ```bash # Use
  timeout /run --timeout 30 slow-command # Run in background with monitoring
  /run --background --monitor npm run dev # Kill hanging processes /kill
  --pattern "hanging-process" /process kill --force process-name ```
</Accordion>

  <Accordion title="Output Buffer Issues">
    **Problem:** Large output causes memory or display issues **Solutions:**
    ```bash # Limit output /run --head 100 command-with-large-output # Stream
    output /run --stream --buffer-lines 50 npm run build # Save to file /run
    --save output.txt command ```
  </Accordion>
</AccordionGroup>
