## Operating Modes Overview

NikCLI operates in four distinct modes, each optimized for different development scenarios. Understanding and effectively using these modes is key to maximizing your productivity.

  </Card>
  <Card title="🖥️ VM Mode" icon="server">

## Mode Switching Commands

### Direct Mode Commands

<Tabs>
  <Tab title="Mode Switching">
    **Direct Mode Commands**
    ```bash
    /default    # Switch to Default Chat mode
    /plan       # Switch to Planning mode
    /auto "task description"  # Switch to Autonomous mode with task
    /vm         # Switch to VM mode
    ```
    
    **Status Commands**
    ```bash
    /stats      # Show usage and session statistics
    /agents     # Show available agents
    ```
  </Tab>
  
  <Tab title="Configuration">
    **View Current Configuration**
    ```bash
    /config     # Show current configuration
    /debug      # Show debug information
    ```

## Planning Mode (📋)

### Planning Workflow

<Steps>
  <Step title="Plan Creation">
    Generate comprehensive implementation plans
    
    ```
    /plan create "User authentication system with social login"

    - Current stack: React + Node.js + MongoDB
    - Authentication needs: Email/password + OAuth
    - Security requirements: JWT tokens, password hashing
    - Integration points: Frontend auth context, protected routes
    
    ### Implementation Phases
    
    #### Phase 1: Backend Foundation (2-3 hours)
    - User model with Mongoose schema
    - Password hashing with bcrypt
    - JWT token generation and validation
    - Authentication middleware
    - Basic login/register endpoints
    
    #### Phase 2: OAuth Integration (1-2 hours)  
    - Passport.js setup for Google/GitHub
    - OAuth callback handling
    - User account linking logic
    - Social profile data integration
    
    #### Phase 3: Frontend Implementation (2-3 hours)
    - Authentication context/provider
    - Login and registration forms
    - Social login buttons
    - Protected route wrapper
    - User profile management
    
    #### Phase 4: Security & Testing (1-2 hours)
    - Input validation and sanitization
    - Rate limiting implementation
    - Unit and integration tests
    - Security audit and hardening
    
    ### Technical Specifications
    - Database: MongoDB with Mongoose ODM
    - Authentication: JWT with refresh tokens
    - OAuth: Passport.js with Google/GitHub strategies
    - Frontend: React Context API for state management
    - Validation: Joi for backend, custom hooks for frontend
    
    ### Dependencies
    - Backend: bcryptjs, jsonwebtoken, passport, passport-google-oauth20
    ```

### Autonomous Mode

Autonomous mode enables AI-driven execution with minimal supervision.

```bash
# Start autonomous execution
/auto "task description"

# Examples
/auto "create a todo application with React and Node.js"
/auto "implement user authentication with JWT tokens"
/auto "optimize application performance"
```

**Autonomous Features:**
- Automatic task execution
- Minimal user intervention
- Progress tracking
- Quality assurance

**Control Commands:**
```bash
# View progress and status
/stats      # Show usage statistics
/agents     # Show agent status

# Safety controls
/dev-mode   # Enable developer mode
/safe-mode  # Enable safe mode
/clear-approvals  # Clear cached approvals
```

### VM Mode

VM mode provides secure, isolated development environments using Docker containers.

```bash
# Switch to VM mode
/vm
/vm-mode

# VM operations
/vm-create https://github.com/user/repo.git  # Create new container
/vm-list                                     # List active containers
/vm-connect [container-id]                   # Connect to container
/vm-create-pr [id] "title" "description"     # Create pull request
```

**VM Mode Features:**
- Docker container isolation
- Repository cloning and analysis
- Secure development environment
- Pull request creation capabilities
- API key security proxy

**VM Management Commands:**
```bash
/vm-stop [container-id]     # Stop container
/vm-remove [container-id]   # Remove container
/vm-logs [container-id]     # View container logs
```

## Mode Comparison

| Feature | Default Chat 💬 | Planning Mode 📋 | Autonomous Mode 🤖 | VM Mode 🖥️ |
|---------|------------------|-------------------|-------------------|------------------|
| **Supervision** | High | Medium | Low | Medium |
| **Speed** | Slow | Medium | Fast | Medium |
| **Learning** | High | Medium | Low | Medium |
| **Complexity** | Simple-Medium | Medium-High | Any | Medium-High |
| **Approval** | Every step | Major milestones | Safety checkpoints | Container operations |
| **Security** | Standard | Standard | Reduced (dev-mode) | Maximum (isolated) |
| **Best For** | Learning, precision | Complex planning | Rapid development | Secure analysis |
