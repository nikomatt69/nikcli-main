# Configuration

NikCLI offers extensive configuration options for models, agents, security, and performance.

## Configuration Commands

```bash
/config            # Show current configuration
```

## Model Configuration

### Switching Models

```bash
/model <name>      # Switch model
/models            # List available models
/temp <0.0-2.0>    # Set temperature
```

### Model Routing

```bash
/router status     # Check router status
/router on         # Enable adaptive routing
/router off        # Disable routing
/router mode balanced  # Set mode: conservative, balanced, aggressive
```

## API Keys

### Setting Keys

```bash
# AI Providers
/set-key anthropic <key>
/set-key openai <key>
/set-key google <key>
/set-key openrouter <key>
/set-key groq <key>
/set-key cerebras <key>

# Additional Services
/set-key-redis          # Configure Redis/Upstash
/set-vector-key         # Configure Upstash Vector
/set-key-bb             # Configure Browserbase
/set-key-figma          # Configure Figma
/set-coin-keys          # Configure Coinbase (Web3)
/set-key-nikdrive <key> # Configure NikDrive
```

## Default Settings

### Model Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `temperature` | `1.0` | Model creativity (0-2) |
| `maxTokens` | `6000` | Max output tokens |
| `chatHistory` | `true` | Enable history |
| `maxHistoryLength` | `100` | Max history messages |

### Security Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `securityMode` | `safe` | `safe`, `default`, `developer` |
| `approvalPolicy` | `moderate` | `strict`, `moderate`, `permissive` |
| `requireApprovalForNetwork` | `true` | Require network approval |

### Session Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `approvalTimeoutMs` | `30000` | 30 second timeout |
| `devModeTimeoutMs` | `3600000` | 1 hour timeout |
| `autoApproveReadOnly` | `true` | Auto-approve reads |

## Security Modes

### Safe Mode (Default)
```bash
/safe-mode
```
Maximum security with approval for all operations.

### Default Mode
```bash
/security set default
```
Balanced security with approval for risky operations.

### Developer Mode
```bash
/dev-mode enable
/dev-mode status
```
Reduced restrictions for trusted development.

<Warning>
Developer mode reduces security checks. Only use in trusted environments.
</Warning>

## Feature Flags

Enable via environment variables:

### Semantic Cache
```bash
export NIKCLI_SEMANTIC_CACHE=true
```
- `minSimilarity`: 0.85
- `ttl`: 3600 seconds
- `maxCacheSize`: 100,000 entries

### Agent Memory
```bash
export NIKCLI_AGENT_MEMORY=true
```
- `maxMemorySize`: 10,000 entries
- `retrievalTopK`: 5
- `autoLearning`: true

### Debug Dashboard
```bash
export NIKCLI_DEBUG_DASHBOARD=true
```
- `updateInterval`: 500ms
- `theme`: `dark` or `light`

## Caching

### Redis/Upstash

```bash
/redis-enable      # Enable Redis
/redis-disable     # Disable Redis
/redis-status      # Check status
```

Environment variables:
```bash
export REDIS_URL="redis://localhost:6379"
export UPSTASH_REDIS_REST_URL="https://..."
export UPSTASH_REDIS_REST_TOKEN="..."
```

### Cache Management

```bash
/cache stats       # Show statistics
/cache clear       # Clear cache
/cache settings    # View settings
```
