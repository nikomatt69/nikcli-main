# NikCLI Quickstart Guide

Get started with NikCLI using real working examples of the most commonly used commands and workflows. This guide includes tested examples you can run immediately.

## Installation & Setup

### Quick Install (Recommended)

<Accordion title="Universal Installation (Any Package Manager)">

<Tabs>

  </Tab>

## Essential Commands & Workflows

### 1. Basic Chat Mode

Start with natural language commands:

```bash
# Start interactive session
nikcli
```

Then in the chat interface:
```
# Create a React component
Create a React component with TypeScript that displays a user profile card

# Analyze your project
Analyze my project structure and suggest improvements

# Generate code
Write a Node.js function to validate email addresses
```

### 2. File Operations

<Accordion title="Reading and Writing Files">

```bash
# Read a file with pagination
/read src/components/Button.tsx

# Read specific lines
/read src/utils/helpers.ts --from 10 --to 25

# Read with more command for large files
/read large-file.log --more

# Write content to a file
/write config.json {"name": "my-app", "version": "1.0.0"}

# Create a new file with content
/write src/types/user.ts "export interface User { id: string; name: string; email: string; }"

# List files in directory
/ls src/components

# Search for content in files
/search "useState" src/

# Find files by pattern
/find-files "*.test.ts"
```

</Accordion>

### 6. Agent System

<Accordion title="Working with AI Agents">


</Accordion>

## Common Workflows

### Workflow 1: Project Setup

<Steps>
  <Step title="Initialize Project">
    ```bash
    # Start NikCLI
    nikcli
    
    # In chat:
    /init
    ```
  </Step>
  <Step title="Create Project Structure">
    ```
    Create a React TypeScript project with the following structure:
    - src/components/ for React components
    - src/utils/ for utility functions  
    - src/types/ for TypeScript interfaces
    - tests/ for test files
    ```
  </Step>
  <Step title="Set Up Dependencies">
    ```bash
    /install react react-dom typescript @types/react @types/react-dom
    /install vite @vitejs/plugin-react
    /install vitest @testing-library/react
    ```
  </Step>
  <Step title="Configure Build System">
    ```bash
    /write package.json
    {
      "name": "my-react-app",
      "version": "1.0.0",
      "scripts": {
        "dev": "vite",
        "build": "tsc && vite build",
        "test": "vitest",
        "lint": "eslint src --ext .ts,.tsx"
      }
    }
    ```
  </Step>
</Steps>

### Workflow 2: Code Review & Analysis

<Steps>
  <Step title="Analyze Project Structure">
    ```bash
    /analyze-project
    /search "TODO" src/
    /find-files "*.ts"
    ```
  </Step>
  <Step title="Review Code Quality">
    ```bash
    /lint
    /run npm audit
    /test
    ```
  </Step>
  <Step title="Get Recommendations">
    ```
    Analyze the codebase and suggest improvements for:
    - Code structure and organization
    - Performance optimizations
    - Security best practices
    - Testing coverage
    ```
  </Step>
</Steps>

### Workflow 3: Feature Development

<Steps>
  <Step title="Switch to Plan Mode">
    ```bash
    /plan
    ```
  </Step>
  <Step title="Generate Feature Plan">
    ```
    Create a plan to implement user authentication with:
    - JWT token management
    - Password hashing
    - User registration and login
    - Protected routes
    - Session management
    ```
  </Step>
  <Step title="Execute Plan">
    ```bash
    # After reviewing the generated plan
    # Select "Yes" to execute
    # Or manually execute steps:
    /write src/auth/jwt.ts "import jwt from 'jsonwebtoken'; export const generateToken = () => {...}"
    /write src/auth/password.ts "import bcrypt from 'bcrypt'; export const hashPassword = async () => {...}"
    ```
  </Step>
  <Step title="Test Implementation">
    ```bash
    /test
    /run npm run test:auth
    ```
  </Step>
</Steps>

## Configuration Examples

### Environment Variables

<Accordion title="Key Environment Variables">

```bash
# API Keys
export ANTHROPIC_API_KEY="your-claude-key"
export OPENAI_API_KEY="your-openai-key"
export GOOGLE_GENERATIVE_AI_API_KEY="your-gemini-key"

# Ollama (Local Models)
export OLLAMA_HOST="localhost:11434"

# Redis Cache
export UPSTASH_REDIS_REST_URL="your-redis-url"
export UPSTASH_REDIS_REST_TOKEN="your-redis-token"

# Compact Mode
export NIKCLI_COMPACT=1
export NIKCLI_CLEAN_CHAT=1

# Token Optimization
export EMBED_BATCH_SIZE=300
export NIKCLI_MINIMAL_STREAM=1
```

</Accordion>

### Configuration File

Create a `nikcli.config.json` in your project root:

```json
{
  "currentModel": "claude-3-5-sonnet-latest",
  "temperature": 0.7,
  "maxTokens": 4000,
  "chatHistory": true,
  "maxConcurrentAgents": 3,
  "securityMode": "default",
  "redis": {
    "enabled": true,
    "host": "localhost",
    "port": 6379
  },
  "supabase": {
    "enabled": true,
    "url": "your-supabase-url",
    "features": {
      "database": true,
      "auth": true,
      "storage": true
    }
  }
}
```

## Troubleshooting

### Common Issues

<Accordion title="Installation Issues">

**Node.js Version Error:**
```bash
# Check Node.js version (requires 18+)
node --version

# Update Node.js if needed
# Use nvm, fnm, or your preferred version manager
```

**Package Manager Issues:**
```bash
# Clear cache and reinstall
npm cache clean --force
npm install -g @nicomatt69/nikcli
```

**Permission Errors:**
```bash
# Use npx instead of global install
npx @nicomatt69/nikcli

# Or fix npm permissions
sudo chown -R $(whoami) ~/.npm
```

</Accordion>

<Accordion title="Connection Issues">

**API Key Problems:**
```bash
# Check if keys are set
/models

# Set keys interactively
/set-key anthropic your-key-here
/set-key openai your-key-here
```

**Network Issues:**
```bash
# Test connection
/status

# Check specific API
/test
```

</Accordion>

<Accordion title="Performance Issues">

**High Token Usage:**
```bash
# Check current usage
/tokens

# Enable compact mode
export NIKCLI_COMPACT=1

# Clear session
/clear
```

**Slow Operations:**
```bash
# Enable Redis caching
/redis-enable

# Check cache status
/cache stats
```

</Accordion>

## Best Practices

### 1. Start Simple
- Begin with basic commands and natural language
- Use `/help` frequently to discover features
- Start with the default agent before creating custom ones

### 2. Use Modes Appropriately
- **Default Mode**: Quick tasks and questions
- **Plan Mode**: Complex multi-step operations
- **VM Mode**: Containerized development

### 3. Security First
- Always review commands before approving
- Use `/security status` to check settings
- Set appropriate approval policies for your team

### 4. Leverage Context
- Initialize projects with `/init` first
- Use `/context` to manage project understanding
- Keep `/NIKOCLI.md` updated for better context

### 5. Monitor Usage
- Regularly check `/tokens` for usage
- Use `/compact` for large projects
- Configure appropriate rate limits
